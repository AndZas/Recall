# Recall Project Rules

These rules define the long-term direction of Recall. Every new feature,
architectural decision and refactoring should follow them.

## 1. Moment First

The primary entity of Recall is a **Moment**, not a media file.

## 2. AI Is a Tool

AI exists only to reduce manual work.

## 3. Offline First

Core functionality should work locally whenever practical.

## 4. Automation First

Automate repetitive work whenever possible.

## 5. Creator First

Every decision should improve the creator's workflow.

## 6. Modular Architecture

Every subsystem should be replaceable through clear interfaces.

## 7. Pipeline-Based Processing

Video → Audio → Speech → Vision → OCR → Metadata → Search Index

## 8. Explainability

Automatic decisions should be understandable and, when possible, include
confidence information.

## 9. No Vendor Lock-in

Avoid hard dependencies on a single AI model or service.

## 10. Practical Features Only

Every feature must save the creator time.

## Definition of Success

Recall succeeds when users stop thinking about files and only think
about the moments they want to find.
